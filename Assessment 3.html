    Name Web Attacks

    Cross-Site scripting

    Source: https://www.cloudflare.com/learning/security/threats/cross-site-scripting/


    According to tripwire, cross-site scripting or an XSS attack tricks a browser into delivering malicious client-side scripts to the victim's browser. Thia malicious script will automatically execute once recieved. These scripts can exfiltrate data, install malware or redirect the user to a spoofed site.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    SQL Injection Attacks

    Source: https://portswigger.net/web-security/sql-injection


    An SQL injection attack is one of the most common web attacks in recent years. This attack aims to compromise cookies, web forms or HTTP posts in a server to manipulate data out of it. These attacks exploit input fields such as text boxes to inject malicious scripts to trick servers into providing unauthorised information.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Broken Authentification

    Source: https://my.f5.com/manage/s/article/K000135769

    		Approximately 67% of data breaches are from compromised credentials. These credentials can be compromised through brute force, credential stuffing, dictionary attacks or many other ways. Broken authentification is simply any illegimate login.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Drive-By Download

    Source: https://us.norton.com/blog/malware/what-are-drive-by-downloads

    		A drive-by download occurs when a user visits a website and said website automatically downloads a malicious agent. Simply visiting a page, clicking a pop up or opening an email can cause one of these attacks.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Password-Based Attacks

    Source: https://us.norton.com/blog/emerging-threats/password-attack

    		Similar to broken authentification exploits, but slightly different. This is where credential dumping such as stealing your RAM, brute forcing again, credential stuffing again or 'Pass the Has' techniques are used to sign in to accounts illegitimately.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Fuzzing

    Source: https://www.synopsys.com/glossary/what-is-fuzz-testing.html

    		To fuzz test is to pump large amounts of random data or fuzz into an application to get it to crash. Then the crash is then analysed to identify weak spots. This allows hackers to find loopholes in security that can be further exploited.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Using Components with Known Vulnerabilities

    Source: https://cheapsslsecurity.com/blog/4-dangers-of-using-components-with-known-vulnerabilities/

    		Software and hardware are both made of a lot of moving parts. A simple exploit used is to identify risks and flaws in parts of open source software or supply chains that you can manipulate to steal data or find vulnerabilities.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    DDoS

    Source: https://www.cloudflare.com/learning/ddos/what-is-a-ddos-attack/

    		DDoS or Distributed Denial-of-Service attacks essentially use a botnet to create vast numbers of requests to a target server in order to make it unavailable for other users. These attacks are often used in conjunction with others to distract security whilst you exploit a vulnerability.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    MiTM

    Source: https://threatcop.com/blog/man-in-the-middle-attack/

    		MiTM or Man in the Middle attacks are mostly used for unencrypted sites. Hackers can intercept the data as it is being trasnferred between parties and if it isn't encrypted the hacker can simply read personal information easily.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself

    Directory Traversal

    Source: https://www.fastly.com/blog/back-to-basics-directory-traversal/

    		Directory/Path Traversal attacks occur when a hacker attempts to access a root folder unauthorised. Eventually by reaching enough folders prior in hierarchy hackers can often access the site, config files, databases or etc.
    https://www.tripwire.com/state-of-security/most-common-website-security-attacks-and-how-to-protect-yourself
